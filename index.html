Quiz App

 Simple Web App

The Quiz App is an intelligence test based on Howard Gardner's test. It tests verbal, numerical, logical, and creative intelligences.

This was an exercise to build an interactive app from scratch using vanilla JS. Some of the technical requirements I explored: rendering answer choices in a <form>, using semantic HTML, along with CSS and jQuery, following a11y best practices, using responsive design and being fully usable by keyboard.

Word App

 API Hack

The Word App helps users get a better understanding and feeling of words. Click a button and the app will generate a random word. That word will be passed through a dictionary, lyric, and poem API. The song title from the lyrics will be passed through YouTube's API to search for a video. The poem title from the poem will be passed through Google Book's API to search for a book.

Everything is passed through a sentiment analysis to get a sentiment score. The sentiment score is coded into the app and doesn't use an API.

This was created an exercise to connect and feed multiple APIs together to build a bigger picture, starting from a single endpoint. A total of six APIs were used to pass data from one to another.

  • CSS
  • Javascript

Project Mech

 Work in progress Full stack project

Project Mech is a community for car owners where users can register an account, save their car make and model, and search diagnostic trouble codes (DTC). After performing the DTC search, users can chat with other car owners for potential solutions.

This was an exercise to create a front-end, back-end, and database. To build this project, it required several different topical explorations, such as JSON web tokens (JWT) for authentication and login, Node JS, Express, and Knex to build the back-end, and Postgresql to create the database. I explored some tools too, like DBeaver used with Postgresql. I also picked up Scrapy using Python to scrape DTC data from the web. Fun times!

I'm looking for collaborators on this project to source more DTC codes. Check the repo for more information.

Thought progress and planning: (1) initial brainstorm, (2) project overview, (3) user flow, (4) user stories, (5) alpha testing outreach template, (6) user survey, (7) survey results.

Project Cellier

 Work in progress  Full stack project

Project Cellier is a search tool that helps people find different distributors of essential oils, fragrance oils, absolutes, and waxes. It will also recommend oils that mix well with others based on a fragrance wheel.

This is another exercise to create a front-end, back-end, and database. I'm focusing more on unit testing on this project, essentially testing all the endpoints and services.

The back-end is about 90% built. I'm thinking of exploring Passport JS for authentication to allow users to only update and delete their own objects. The front-end is currently only a shell.

Thought progress and planning: (1) initial brainstorm, (2) MVP scent database structure and tables, (3) MVP user flow, (4) MVP user stories.
