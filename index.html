PROJECT MECH
 Full Stack

Repository

Work in progress...
Overview
Project Mech is a community for car owners where users can register an account, save their car make and model, and search diagnostic trouble codes (DTC). After performing the DTC search, users can chat Reddit-style with other car owners for potential solutions.

Learnings
This was an exercise to create a front-end, back-end, and database. To build this project, it required several different topical explorations, such as JSON web tokens (JWT) for authentication and login, Node, Express, and Knex to build the back-end, and Postgresql to create the database. I explored some tools as well, like DBeaver used with Postgresql. I also picked up Scrapy using Python to scrape DTC data from the web. Fun times!

I'm looking for collaborators on this project to source more DTC codes. Check the repo for more information.

PROJECT CELLIER
 Full Stack

Repository

Work in progress...
Overview
Project Cellier is a search tool that helps people find different distributors of essential oils, fragrance oils, absolutes, and waxes. It will also recommend oils that mix well with others based on a fragrance wheel.

Learnings
This is another exercise to create a front-end, back-end, and database. I'm focusing more on unit testing on this project, essentially testing all the endpoints and services. Currently there are about sixty-four tests in place.

The back-end is about 90% built. I'm thinking of exploring authentication to allow users to only update and delete their own objects. For example, each comment is saved with a users_id as one of the columns. When a user logs in, he is given his own token. The token expires based on inactivity. When a user tries to delete or patch his own comment, the particular comment is verified using a users_id that is in the token and the users_id of the comment. The front-end is currently only a shell.

WORD APP
 Vanilla Javascript

Repository

Overview
The Word App helps users acquire a better understanding and feeling of words through definitions, poems, lyrics, videos, and sentiment scores. Click a button and the app will generate a random word. That word will be passed through a dictionary, lyric, and poem API. The song title from the lyrics will be passed through YouTube's API to search for a video. The poem title will be passed through Google Book's API to search for a book. Everything is passed through a sentiment analysis to get a sentiment score. The sentiment score is coded into the app and doesn't use an API.

Learnings
This was created as an exercise to connect and feed multiple APIs together to build a bigger picture starting from a single endpoint. I learned how to feed multiple APIs from an output from a root API. A total of six APIs were used to pass data from one to another.
